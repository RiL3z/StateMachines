Nondeterministic Finite Automaton Abstract Data Type

The NFA (nondeterministic finite automaton) can be thought of as a simple machine that can determine if input strings belong to languages that it is describing. It exhibits parallelism in that the machine can take multiple branches in determining if an input string belongs to the language.

Formally speaking, an NFA consists of 5 major parts:

  • A finite set of states.
  • A finite alphabet that the machine can read symbols from.
  • A transition function that decides how the machine transitions from state to state.
  • One starting state that the machine begins execution in.
  • A set of final states. If the machine is in any one final state when all the input is read, the machine accepts the string as belonging to the language it describes.

From this description, an abstract data type can be imagined that supports the following methods:

  • addState(label): Adds a state to the machine with the given label. Labels should be unique so one state is always distinguishable from another.
  • addTransition(state, symbol, nextState): Adds a transition from one state to another.
  • addFinalState(label): Adds a final state to the machine.
  • NFA(startLabel): Creates an NFA object with the start state labeled with startLabel.
  • test(expression): Runs the machine on the expression string in order to determine if it is acceptable or not.
