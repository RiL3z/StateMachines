State Machines

Summary

Build an object oriented library of data structures and algorithms that facilitates the modelling of state machines, the conversions from one type of state machine to another, and basic visualization of those state machines.

Background

I can envision applying parallism to the functioning of NFAs (Non-deterministic Finite State Machines), and also applying parallism to the DFA (Deterministic Finite State Machines) <=> NFA <=> Regular Expression conversion algorithms. Visualizing the state machines may also benefit from parallization.

The Challenge

Understanding the fine details of state machines in a theoretical sense is challenging, but also figuring out a way to parallize the string processing (accept or reject) of NFA's and all the conversion algorithms will be challenging. I also think that I will be using a parallel platform that I am not familiar with, so learning the ropes of that will also be a challenge.

Resources

I plan on starting the codebase for this project from scratch. I am using a book that I own called "Introduction to the Theory of Computation" by Michael Sipser as my main reference for understanding the workings of state machines and the algorithms that go along with them.

Goals and Deliverables

  • Goals

    • Plan to Achieve

      I plan to definitely build a library with at least NFA's as a data structure and to utilize parallel processing in the algorithm that decides whether a test string belongs to the langauge the machine is describing (accept or reject). I believe I can achieve this because I have a good understanding of Abstract Data Types and how to implement them. I also have enough displine and focus to read a theoretical text and understand what's in it and what it is trying to say.

    • Hope to Achieve

      • I hope to achieve the parallel DFA <=> NFA <=> Regular Expression conversion algorithms in a parallel fashion, insofar as that is possible.
      • I also hope to achieve basic drawing and state machine visualization

  • Deliverables

    • Class/Object library with DFA, NFA, and Regular Expression data structures and algorithms.
    • Application Programmer Interface Document that informs programmers on the use of the library.

Platform Choice

This requires more study on my part. I want a programming language and a supporting platform/library that supports object orientation along with parallism on a multitude of parallel-enabled architectures. I think I can use c++ along with pthreads to address that problem.
